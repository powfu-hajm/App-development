## 青心伴App UI设计与前端开发概述(1-5 weeks)
### 1.目前已初步完成设计的界面
- 登录与注册界面（LoginActivity&RegisterActivity）
- 主界面（MainActivity&HomeFragment）
- AI对话界面（ChatFragment）
- 心情日记界面(DiaryFragment)
- 个人信息界面(ProfileFragment)
- 心理测试界面(TestFragment)

<span style="color:red">目前所有界面的类文件com.example.app包下，后期将整合到其二级子文件夹/包ui下,同时ui下又对各个界面作出了分类存储（又建立了各自的子文件夹/包）。</span>
### 2.当前页面跳转逻辑
主体的跳转逻辑为:

- （登录注册界面）→ 主界面 → 其他界面（通过主界面导航栏跳转）
- 个人信息界面 → 登录界面
由于我们目前没有连接数据库，因此App运行后，我们目前默认让其跳转到主界面（即跳过登录与注册界面，可在AndroidManifest.xml文件中更改App首次进入的界面），但可于个人信息界面点击退出登录后跳转到登录界面（点击注册按钮也可跳转到注册界面），所有的输入框可正常输入信息但APP目前无存储功能。
### 3.辅助类
在java文件中有一些类是辅助类非直接与某一xml文件一一对应，目前有：
- 对应于AI交流界面：ChatAdapter、ChatMessage、ChatViewModel
- 对应于心情日记界面：DiaryEntry、DiaryViewModel
- 对应于个人信息界面：ProfileViewModel
- 对应于心理测试界面：TestViewModel

<span style="color:red">这些文件大部分是用于界面在运行是的辅助工具，但大多数都为模拟实现，之后在开发时可删改。</span>
### 4.布局设计
目前在res文件夹下，各个文件夹存储的内容具体为:
- drawable：存放待使用图标<span style="color:red">（目前图标在引入时有误，于是使用内置图标暂时代替）</span>、导航样式和按钮样式等
- layout：存放各个界面实现的布局文件,对应于前面的每一个界面
- menu：导航栏实现
- mipmap：存放图标（暂未使用）
- values/themes.xml：主题设置（颜色常量设定-colors.xml、样式设定-styles.xml、strings.xml-字符串常量设定）

### 5.前期开发时遇到的问题
- 首次运行项目时，gradle工具可能会报错，目前应该已经修改完毕。（不太确定？再运行试试？）
- 传递项目时要传递完整，如若只传递app子文件夹而不包含gradle工具相关的配置文件等，则运行项目时会报错。
- 早期的项目开发语言是kotlin和java混合的形式，目前已经修改为纯java项目。
- 早期的项目较为凌乱（有些界面仅用于在早期开发时测试使用，后期则没必要保留），不便于小组成员阅读，目前已调整，将非必要的界面已经删除，并统一了命名，且保证一个界面有一个java与xml文件对应。

### 6.注意事项
- 当前许多界面的图标都尚未完善，有一部分是以安卓默认图标代替（小机器人），有一部分是利用Android Studio内置图标进行替代。
- 当前大多的按钮逻辑都是简单事件监听，除一部分可做到界面跳转外，其他在点击时只能给出点击提示，还有一些点击可能会退出App。
- 当前App可下载到手机中调试。（如果时安卓手机的话）
- 界面配色与风格后期可能还会调整。
- 如果App打开即出现闪退，请检查主界面xml代码是否有误。
- 目前项目中搭建了初步的前端与后端框架但并未投入使用，后面可根据需要进一步修改。
- 如果还出现了其他问题或对项目存在疑问，随时交流！！！
